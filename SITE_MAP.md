# 루모스(Rumos) 사이트맵

요구사항 기준. 엑셀 레퍼런스가 있으면 메뉴명·순서는 해당 문서와 맞출 것.

---

## 상단 네비게이션

- **호버 → 전체 사이트맵:** 상단 1차 메뉴에 마우스를 올리면 **전체 사이트맵을 보여 주는 패널**(메가 메뉴·풀폭 드롭다운 등)이 **펼쳐짐**.
- **호버 효과:** 메뉴·하위 링크 강조는 **파란색 계열**로 통일(톤앤매너와 일치).
- **하위 링크:** 패널에 나열된 **하위 사이트맵 항목을 클릭**하면 해당 URL로 **이동**.
- **i18n:** 상단바 우측(또는 로고 반대편)에 **언어 전환 버튼** 배치(예: `KO | EN`, 지구본 아이콘+드롭다운 등). 라우팅·저장 방식은 아래 **i18n** 절과 연동.

*(키보드·포커스·모바일에서는 호버 대신 탭/햄버거로 동일 구조 제공 권장.)*

---

## 메인페이지 (`/`)

### Section 1 — 히어로

- 대표 이미지 **또는** 핵심 문구 (콘텐츠 제공 예정, **젠텍 참고**)

### Section 2 — 브랜드 스토리

- 루모스 **어원** 및 **배경 설명**

### Section 3 — Product

메인에 프로덕트 섹션을 두고, 각 항목 클릭 시 **상세 페이지**로 이동.

| 번호 | 프로덕트 |
|------|----------|
| 3.1 | LUM-B |
| 3.2 | LUM-B-L |
| 3.3 | LUM-F |
| 3.4 | LUM-Z |
| 3.5 | Software |

*(상세 URL 예: `/products/lum-b`, `/products/software` 등 slug로 통일)*

메인 외에도 오시는 길·Application 등 기존 내비·섹션 구성은 아래 **사이트맵**과 동일하게 유지.

---

## 사이트맵

```
메인페이지 (/)
│
├── [상단 NAV]
│
├── 오시는 길
│   └── 찾아오는 길 섹션 포함 (지도 API 등)
│
├── Application
│   ├── 한 페이지에 항목을 나열하는 형태
│   ├── 항목별 상세 페이지는 없어도 됨
│   ├── 항목당: 사진 1장 + 설명 약 5가지 (예: 반도체, NAND 등)
│   └── 원하는 이미지·소개 자료는 별도 제공
│
├── Product
│   ├── 메인 Section 3 및 필요 시 목록 페이지
│   ├── 3.1 LUM-B · 3.2 LUM-B-L · 3.3 LUM-F · 3.4 LUM-Z · 3.5 Software
│   └── 각 프로덕트 클릭 → 상세 페이지
│
└── Contact (하위 구조)
    ├── Contact Us (/contact/…)
    │   ├── 연락처 + 문의 폼 (제공 예정, 젠텍 참고)
    │   ├── 문의는 이메일 연동 예정
    │   └── 문의 수신 시: 이메일 전달 + 관리자 페이지 누적
    │
    └── 자료실 (/contact/…)
        ├── 자료실 목록
        └── 자료실 상세 페이지 (별도 구성)
```

### URL 예시 (구현 시)

| 화면 | 경로 예시 |
|------|-----------|
| 메인 | `/` |
| 오시는 길 | `/directions` 또는 메인 `#directions` |
| Application | `/applications` |
| Product 상세 | `/products/[slug]` (LUM 계열·Software 각각) |
| Contact Us | `/contact` 또는 `/contact/us` |
| 자료실 목록 | `/contact/resources` |
| 자료실 상세 | `/contact/resources/[id]` |

Contact를 단일 랜딩(`/contact`)으로 두고 하위만 드롭다운·탭으로 나누는 방식도 가능.

## 관리자 페이지

- **Application:** 관련 **이미지·문구** 수정 가능.
- **관리 대상:** **메인 페이지**, **Application**, **Product**, **자료실**, **문의** 등.
- **Product:** 상세 페이지 관리 및 **신규 프로덕트·페이지 추가** 가능하도록 구성.
- **인프라:** 에셋·파일 관리를 위해 **DB 구축** 및 **서버 구성** 필요 (메타데이터 + 스토리지 연동 등 별도 설계).

법적 페이지: 개인정보처리방침·이용약관 등은 푸터 링크로 추가 가능.

## i18n

- **UI:** 상단 네비게이션에 **언어 버튼**으로 노출(위 **상단 네비게이션** 절 참고).
- **구현:** 경로 접두(`/ko`, `/en` 등) 또는 쿼리 방식 중 프로젝트에서 통일.
